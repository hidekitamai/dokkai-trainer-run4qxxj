# 引き継ぎ文書：アソシエイト育成・成長度可視化プロジェクト

作成日：２０２６年７月１２日
前セッション：Claude Code on the web（本文書と成果物ＨＴＭＬは同ディレクトリに保存）

## １　背景と目的

- 弁護士法人親和法律事務所（東京・大阪・松山の３拠点）。依頼者はパートナー弁護士（tamai@shinwalaw.jp）。
- アソシエイト２名の教育のためにClaude Codeを活用したい、というのが出発点。
- 検討の結果、「アソシエイトの成長度を測る仕組み」を作る方向で合意。最終的な分析レポートのデザイン試作まで完了した。
- 具体的な仕組みの作り込みは、依頼者のパソコン側のセッションで行う予定（本文書はその引き継ぎ用）。

## ２　合意済みの設計方針

### （１）三層構えの評価スキーム

1. **第１層：活動量の可視化**──案件管理表（スプレッドシート）で、終結件数、起案数（種類別）、期日出頭数、単独相談対応数などを記録。難易度（Ａ〜Ｃ）を１列設けて単純比較の弊害を緩和する。
2. **第２層：起案添削の差分分析（本命）**──アソシエイトの第一稿とパートナー添削後の最終稿をＡＩが比較し、指摘を６区分（誤字・書式／文章表現／構成・論理／要件事実の過不足／論点の見落とし／事実摘示の不正確）に分類・集計。「どの区分の指摘が減っていくか」を成長曲線として追う。
3. **第３層：ルーブリック評価（四半期面談用）**──依頼者が特に採用を決めた部分。５指標×５段階。
   - ５指標：**起案力・調査力・依頼者対応・期日対応・自律性**
   - 尺度：１＝常時指導を要する、３＝概略の指示で処理できる、５＝単独処理を任せられる
   - 位置づけ：ＡＩ分析（第１・２層、Slack分析）は参考資料にとどめ、評点は面談を経て評価者（パートナー）が確定する。

### （２）Slack分析（調査力・依頼者対応の材料）

- 実現方法は２つ。**Ａ案：claude.aiのSlackコネクタを接続**して直接分析（四半期ごとの定期運用向き。前セッション時点では未接続）。**Ｂ案：Slackエクスポートを取り込んで分析**（過去分の一括分析向き。標準プランのエクスポートはパブリックチャンネルのみ、ＤＭは上位プラン＋手続が必要な点に注意）。
- 調査力は、調査依頼への一次回答の速さ、根拠（条文・裁判例・文献）の付し方、行き詰まった際の代替提案などを観察事実として引用付きでレポート化する。
- 依頼者対応は、Slackからは社内報告の質という間接指標しか取れない。直接見るならメール（Gmail／Outlookコネクタ）や匿名化した相談メモが素材候補。
- 運用上の必須条件：**本人への事前説明と同意**（監視ではなく育成目的であることの明示）、**守秘義務対応**（学習に使われない法人プランで実施、レポートには依頼者名を残さない、引用は匿名化）、調査依頼用チャンネルなどの置き場所の整備。

### デザイン仕様（変更時はこれを維持すること）

- 構成：表題→前書き表（対象者・評価期間・資料・作成）→第１総評→第２ルーブリック評価（前期・今期の点比較チャート）→第３起案添削分析（数値＋濃淡バーの表）→第４活動量（４枚のスタット）→第５観察事実（Slack引用付き）→第６次期の重点課題→脚注
- 書体：見出しは明朝（Yu Mincho／Hiragino Mincho ProN系）、本文はゴシック。**数字は全角**、読点は「、」（事務所の書式ルール）。
- 配色：藍色一色の濃淡のみ。ライト＝前期 #8fadc9／今期 #2f5f88（地 #f8f8f6）、ダーク＝前期 #3f5c76／今期 #8fb6d9（地 #171b20）。ダークモード対応済み。色覚・コントラスト検証済み（順序ランプとして合格）。
- 装飾の禁止事項：グラデーション、絵文字、角丸カード、色数の追加。
- 図表要素にはホバーのツールチップあり（data-tt属性）。

## ４　次セッションでやること（優先順）

1. 案件管理表（第１層）の様式を確定し、スプレッドシートの雛形を作る。
2. 添削差分分析（第２層）のパイプラインを試作する。過去の起案（第一稿＋最終稿、匿名化済み）１〜２件で指摘分類レポートを出し、精度を確認する。
3. Slackコネクタの接続（管理者承認が必要な場合あり）と、アソシエイト１名×直近１〜２か月分での「調査力レポート」試作。
4. レポート生成の自動化：上記データを `associate-report.html` の書式に流し込むスクリプト化。事務所名の差し込み（現状は「◯◯法律事務所」のプレースホルダ）。

## ６　書式ルール（このプロジェクトの全成果物に適用）

- 読点は「、」。ただし数字の桁区切りは「,」のまま。
- アラビア数字は全角。
- 計算式を含む回答は、必ずPythonで検算し結果を提示する（本文と検算が食い違う場合は勝手に直さず依頼者に確認する）。
- 裁判関係書類は１００％の書式を目指し、不備は指摘、不明点は質問する。
