# Android → PC 調査結果 自動同期フロー 設計・構築ガイド

> 目的: スマホ(Android)で行った調査・メモ・成果物を、**メールを介さず**、
> ファイル化 → 特定フォルダ保存 → クラウド自動アップロード → PC 同期、という
> **一気通貫のフロー**で即座に活用できる体制を作る。

このドキュメントは「仕組みの設計図」です。実際の設定はスマホ側アプリ(MacroDroid /
Tasker)とクラウド同期クライアントで行いますが、その具体的な手順・注意点をここに集約します。

---

## 1. 現状の課題と目標

| 項目 | 現状 | 目標 |
| --- | --- | --- |
| 転送手段 | メールで PC 宛に送る | クラウド経由で自動同期 |
| 手作業 | 都度、添付・送信・保存 | 保存するだけで PC に到達 |
| 反映速度 | メールを開いて手動保存 | 数十秒〜数分で自動反映 |
| 冗長性 | メールボックス頼み | クラウド + ローカル二重化 |

**設計原則**

1. **人が触るのは「保存」まで**。それ以降(アップロード・同期)は全自動。
2. **単一の入口フォルダ**に集約する。あちこちに散らばらせない。
3. **冗長化**する。クラウド1系統だけに依存しない。
4. **セキュリティ**を最初から織り込む(暗号化・アクセス制御)。

---

## 2. 全体アーキテクチャ

```
[スマホで調査]
      │  (テキスト/写真/PDF/音声メモ など)
      ▼
[ファイル化]  ── 共有メニュー or 自動化アプリが受け取り、命名規則でファイル化
      │
      ▼
[入口フォルダ]  例: /storage/emulated/0/Sync/Inbox/
      │  ← ここに入った瞬間が「自動化の起点」
      ▼
[自動化アプリ]  MacroDroid / Tasker がフォルダ監視 → 整理・リネーム・振り分け
      │
      ▼
[クラウド同期フォルダ]  Google Drive / Dropbox / OneDrive のローカルフォルダ
      │  ↑↓ 双方向同期(自動)
      ▼
[クラウド]  ────────────── 冗長コピー(別クラウド or 端末内バックアップ)
      │
      ▼
[PC 同期クライアント]  Drive/Dropbox デスクトップ版が自動ダウンロード
      │
      ▼
[PC の作業フォルダ]  即座に開いて活用
```

ポイントは **「入口フォルダ」= クラウド同期フォルダの中**にしてしまえば、
自動化アプリなしでも最小構成が成立することです(下記 3-A)。
自動化アプリは「命名・振り分け・通知・トリガー」を足すための**補完**として使います。

---

## 3. 実装パターン(小さく始めて拡張)

### 3-A. 最小構成 — 自動化アプリ不要(まずここから)

1. スマホに **Google Drive アプリ**(または Dropbox / OneDrive)を入れる。
2. Drive アプリの設定で **「フォルダのバックアップ/同期」**を有効化、または
   Drive 上に `Inbox` フォルダを1つ作る。
3. 調査結果は共有メニューから **「Drive に保存」→ `Inbox`** を選ぶだけ。
4. PC 側に **Google ドライブ(パソコン版)** をインストールし、`Inbox` を
   ミラーリング設定にする。
5. → スマホで保存した数分後、PC の `Inbox` に自動で現れる。**メール不要**。

この時点で「メール脱却」は達成できます。次の 3-B / 3-C は快適さ・自動度の上乗せです。

### 3-B. 中間構成 — MacroDroid でフォルダ監視 + 整理

MacroDroid は初心者向けで GUI が分かりやすく、この用途に十分です。

**マクロ例: 「Inbox に入ったファイルを日付フォルダへ振り分け」**

- **トリガー(Trigger)**: `File Modified/Added` → 監視パス `/…/Sync/Inbox/`
- **アクション(Actions)**:
  1. `File Operations → Move` : 移動先を `/…/Sync/Sorted/{year}-{month}-{day}/`
     - MacroDroid の変数 `{dayofmonth}` `{month}` `{year}` でフォルダ名を動的生成。
  2. (任意)`Rename` : `research_{year}{month}{day}_{hour}{minute}.拡張子`
  3. `Notification` : 「調査ファイルを同期キューに追加しました」と通知。
- 移動先 `Sorted/` を**クラウド同期フォルダ内**に置けば、そのままアップロードされる。

**マクロ例: 「クリップボードを1タップでメモ化」**

- **トリガー**: ホーム画面ウィジェット(MacroDroid のショートカット)。
- **アクション**: `Write to File` で `clipboard.txt` を `Inbox/` に追記保存。
  → コピーした調査断片を、アプリを切り替えずにファイル化できる。

### 3-C. 高度構成 — Tasker(細かい制御・条件分岐)

Tasker はより自由度が高く、以下のような込み入った処理に向きます。

- **条件付き振り分け**: ファイル種別(画像/PDF/テキスト)ごとに別フォルダへ。
- **位置・時間トリガー**: 「自宅 Wi‑Fi 接続時だけ大容量ファイルを同期」など、
  通信量・バッテリーに配慮した同期。
- **HTTP 連携**: クラウド API を直接叩いてアップロード、完了を PC に Webhook 通知。
- **AutoTools / AutoShare 等のプラグイン**併用で共有メニューを完全自動化。

> 目安: まず 3-A で「同期そのもの」を成立させ、手間を感じる部分だけ 3-B/3-C で
> 自動化を足す。最初から Tasker を作り込みすぎないのがコツ。

## 4. クラウドの選び方と冗長化

| サービス | 長所 | 注意点 |
| --- | --- | --- |
| Google Drive | Android と親和性が高い、無料15GB | 写真は Google フォトと容量共有 |
| Dropbox | 同期が速く安定、選択同期が優秀 | 無料枠が小さい(2GB) |
| OneDrive | Windows PC と統合が滑らか | Android の自動化はやや不得手 |

**冗長化の具体策(いずれか1つでも実施)**

1. **二系統クラウド**: 入口フォルダを Drive と Dropbox の**両方**に同期
   (MacroDroid で `Move` を2回、または各アプリの同期フォルダにコピー)。
2. **端末内バックアップ**: `Sorted/` を月次で ZIP 化し `Backup/` に退避。
3. **PC 側スナップショット**: PC に届いた `Inbox` を定期的に別ドライブへコピー
   (Windows のファイル履歴 / robocopy / rsync 等)。
4. **世代管理**: クラウドのバージョン履歴機能を有効化(誤上書き対策)。

## 5. セキュリティ

- **暗号化**: 機微な調査結果は保存前に暗号化。
  - 手軽: パスワード付き ZIP(7-Zip / MiXplorer 等)。
  - 本格: Cryptomator でクラウド同期フォルダをまるごと暗号化ボリューム化。
- **アクセス制御**: クラウドアカウントに**2段階認証(MFA)**を必須化。
  共有リンクは作らない/作っても期限・権限を絞る。
- **端末保護**: 画面ロック + 端末暗号化 + 紛失時のリモートワイプ(デバイスを探す)。
- **最小権限**: 自動化アプリに与えるストレージ権限は必要フォルダに限定。
- **ログの扱い**: 通知やクリップボード連携に個人情報が残らないか確認。

---

## 6. OS アップデート・運用上の注意

- **バックグラウンド制限**: Android は省電力のため自動化アプリを止めがち。
  設定 → アプリ → 該当アプリ → **バッテリー最適化を「無効(制限しない)」**に。
- **OS メジャーアップデート後**: フォルダ監視・共有メニュー連携が仕様変更で
  動かなくなることがある。更新後は**必ずテスト実行**(§7)を回す。
- **ストレージ権限モデルの変化**: 新しい Android では「すべてのファイルへの
  アクセス」権限が別途必要な場合がある。監視パスが `Android/data` 配下だと
  制限が強いので、`/storage/emulated/0/Sync/` のような**共有領域**を使う。
- **アプリの自動更新**: 自動化アプリ/同期アプリの更新でマクロが影響を受けたら、
  §7 のチェックで早期発見する。

- [ ] 反映までの所要時間が許容範囲(目安: 数十秒〜数分)
- [ ] 機微ファイルが暗号化されている
- [ ] 端末再起動後もマクロが自動で有効化される

## 8. 命名規則(推奨)

一貫した命名は PC 側での検索・並べ替えを劇的に楽にします。

```
research_YYYYMMDD_HHmm_<キーワード>.<拡張子>
例: research_20260715_0930_契約書レビュー.pdf
```

- 先頭を `research_` に固定 → PC で一括抽出しやすい。
- 日付を `YYYYMMDD` にすると**ファイル名順=時系列順**になる。
- キーワードは任意。省略時は自動化アプリで `untagged` を補う。

## 9. このリポジトリとの関係(応用メモ)

本リポジトリ(読解トレーナー)では、学習結果を Google フォームへ送信して
記録しています。上記フローの発想は、こうした「スマホで生まれたデータを
自動でクラウド → PC/集計基盤へ流す」という点で共通です。
将来的に、フォーム回答をスプレッドシート経由で自動集計・PC 反映する仕組みへ
展開する場合も、本ガイドの「入口を一本化 → 自動同期 → 冗長化 → セキュリティ」
という設計原則がそのまま応用できます。
